# superpowers

> A complete software development methodology for AI coding agents.

## Overview

Superpowers is a set of 14 composable skills and instructions that make AI agents follow proper engineering practices — plan before coding, test before shipping, review before merging.

## 14 Skills

| # | Skill | Purpose |
|---|-------|---------|
| 1 | **brainstorming** | Explore requirements before coding |
| 2 | **writing-plans** | Write implementation plans |
| 3 | **executing-plans** | Execute plans systematically |
| 4 | **subagent-driven-development** | Use sub-agents for tasks |
| 5 | **systematic-debugging** | Debug with root cause analysis |
| 6 | **test-driven-development** | Write tests before code |
| 7 | **verification-before-completion** | Verify before claiming done |
| 8 | **requesting-code-review** | Structure code review requests |
| 9 | **receiving-code-review** | Process review feedback |
| 10 | **dispatching-parallel-agents** | Parallel task execution |
| 11 | **finishing-a-development-branch** | Clean branch completion |
| 12 | **using-git-worktrees** | Use git worktrees |
| 13 | **using-superpowers** | Base usage instructions |
| 14 | **writing-skills** | Write custom skills |

## Philosophy

1. **Plan before code**: Write specs, get approval, then implement
2. **Test before ship**: TDD, verification, edge cases
3. **Review before merge**: Code review is mandatory
4. **Iterate in small steps**: Break big tasks into smaller ones

## Usage

```bash
# Start with brainstorming
superpowers/brainstorming

# Write plan
superpowers/writing-plans

# Execute
superpowers/executing-plans

# Verify
superpowers/verification-before-completion
```
